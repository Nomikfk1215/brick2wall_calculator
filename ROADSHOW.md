# 路演演示脚本

## 一条故事线

客户同学正在做一个砖墙排布计算器。这个工具要根据墙体长、高、厚，输出灰缝尺寸、砖块数量和排布图。项目已经有最小 HTML 展示端，但他第一次用 ByteMind，不知道怎么理解仓库、规划大功能、处理计算 bug。

## Act 1：当前代码仓库理解

讲法：

> 这位同学根据文档站安装并配置 ByteMind，选择合适模型。然后他把自己的 GitHub 仓库交给 ByteMind，先不改代码，只让它理解项目边界。

演示提示词：

```text
请先理解这个砖墙排布计算器仓库，不要修改文件。
请说明项目结构、输入校验、灰缝计算、砖块排布计算、页面渲染分别在哪里。
如果我要增加 3D 自由视角排布预览，应该在哪些模块上扩展？
```

可以让 ByteMind 关注：

- `index.html`
- `src/app.js`
- `src/calculator.js`
- `src/renderer.js`
- `tests/calculator.test.js`

## Act 2：Plan -> Build -> 权限控制 -> 3D 功能开发

讲法：

> 客户同学发现 2D 排布图不够直观，希望加入 3D 自由视角预览。ByteMind 先进入 Plan 模式说明改动范围、依赖选择和验证方式，经用户确认后再 Build。修改文件、安装依赖或运行命令时展示权限控制。

演示提示词：

```text
请读取 issues/feature-3d-wall-preview.md。
我希望为砖墙计算器增加 3D 自由视角排砖预览。
请先进入 Plan 模式，说明你会修改哪些文件、是否需要引入依赖、如何验证，不要直接修改。
```

确认计划后：

```text
同意这个计划，请进入 Build，实现 3D 预览功能，并保持最小依赖。
```

建议演示点：

- Plan 中说明核心计算不重写，只新增 3D 渲染层。
- Build 中展示写文件、运行测试或启动本地服务的权限控制。
- 功能完成后打开页面，展示旋转、缩放、半砖颜色和指标同步。
- 由于仓库刻意保留了 debug issue，Act 2 验收可以只跑与 3D 渲染相关的定向检查；全量 `npm test` 留到 Act 3。

## Act 3：Issue -> Debug -> Commit

讲法：

> 功能展示后，客户同学又拉了一个 GitHub issue：半砖数量为奇数时，页面显示的采购总砖数偏少。ByteMind 先 review issue，再复现失败测试，最后做最小修复并提交。

演示提示词：

```text
请读取 issues/issue-brick-count.md，像接到 GitHub issue 一样先 review。
然后运行测试复现问题，定位为什么采购总砖数偏少，做最小修复，重新测试。
最后给出适合的 commit message。
```

## 预期修复方向

`src/calculator.js` 当前把两个半砖合并为一块整砖时用了向下取整。半砖数量为奇数时，应该向上取整，避免采购数量少算一块。
